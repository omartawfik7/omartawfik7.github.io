ReviewGuard is a full-stack data science and machine learning application that detects fraudulent Amazon product reviews using natural language processing and supervised learning. The system covers the complete ML workflow — automated dataset ingestion via the Kaggle API, a multi-stage NLTK preprocessing pipeline, behavioral feature engineering, multi-model training with cross-validation, REST API deployment, and an interactive editorial-style Flask dashboard with real-time Chart.js analytics.

  • ETL pipeline: Built a full NLTK preprocessing chain including lowercasing, URL and HTML stripping, stopword removal (with negation preservation), Porter Stemming, and WordNet Lemmatization — producing clean, consistent vocabulary across 40,000 real Amazon reviews ingested automatically via kagglehub.
  • 18 behavioral + linguistic KPIs engineered from raw text: sentiment extremity (VADER compound), extreme superlative word density, capitalization ratio, lexical diversity (type-token ratio), exclamation count, punctuation ratio, and a rating–sentiment mismatch gap that flags reviews where star rating contradicts written tone — concatenated with a 10,000-feature TF-IDF unigram/bigram matrix using sublinear TF scaling.
  • Trained and evaluated 4 classifiers — Logistic Regression, Random Forest (200 estimators), Linear SVM, and Gradient Boosting — with ROC-AUC, weighted F1, precision, recall, and 5-fold stratified cross-validation; auto-selected and persisted the best model artifacts via pickle for zero-latency inference.
  • Achieved 90.68% accuracy and 90.62 weighted F1-score on a stratified held-out test set; Flask REST API returns prediction label, fake/genuine probability scores, top TF-IDF signal terms, and per-signal fraud severity breakdowns in real time.
  • Deployed an editorial-style interactive dashboard (named ReviewGuard) featuring a live news ticker, model comparison card grid, animated confusion matrix, probability bars, feature value visualization, and product image mosaic — built with vanilla HTML/CSS/JS and Chart.js with zero external UI frameworks.

DocMind — RAG Document Intelligence Pipeline

An end-to-end Retrieval-Augmented Generation (RAG) pipeline enabling intelligent querying over unstructured documents. Upload any PDF, TXT, or Markdown file — DocMind chunks, embeds, and indexes the content locally, then retrieves the most semantically relevant passages to generate precise, cited answers via a Groq-hosted LLaMA-3.3-70B LLM. Designed to never hallucinate outside the source material.

  • Modular pipeline: document ingestion → 512-char chunking with 80-char overlap → local embeddings (all-MiniLM-L6-v2) → ChromaDB persistent vector store → cosine similarity retrieval of top-6 chunks → Groq LLaMA-3.3-70B generation.
  • Source citations on every answer with relevance % scores and expandable passage excerpts; persistent vector store survives app restarts with duplicate detection preventing re-indexing.
  • Full Flask REST API with drag-and-drop upload, multi-turn conversation history, per-document or cross-document querying, document deletion, and live model health endpoint.
